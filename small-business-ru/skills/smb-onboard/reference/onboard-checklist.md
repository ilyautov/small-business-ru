# Чек-лист онбординга

## Пять вопросов интервью

Задавайте по одному. Дождитесь полного ответа перед следующим. Один уточняющий
вопрос — норма, если ответ размыт; глубже не копайте.

1. **Сфера и тип бизнеса.** «Чем занимается ваш бизнес? Дайте одну фразу».
2. **Размер команды.** «Сколько человек работает с вами, включая вас?»
3. **Три главные головные боли.** «Какие три вещи сейчас больше всего съедают время
   или не дают спать?»
4. **Какими инструментами уже пользуетесь.** «Чем пользуетесь изо дня в день? 1С,
   Битрикс24, Яндекс 360, ЮKassa, Telegram…»
5. **Удобный ритм.** «Как мне выходить на связь — ежедневно, еженедельно или только
   по запросу?»

Если владелец спешит — сократите до вопросов 1, 3 и 4: они кормят больше всего
скиллов дальше.

## Приоритетная матрица коннекторов

Сопоставьте названную головную боль с двумя лучшими коннекторами для старта.
Конкретные сервисы — примеры; в логике используются категории (`~~бухгалтерия`,
`~~платежи` и т.д.).

| Главная боль | Первый коннектор | Второй коннектор | Рецепт «показать пользу» |
|---|---|---|---|
| Деньги / счета | `~~бухгалтерия` (1С, МойСклад) | `~~платежи` (ЮKassa, Тинькофф Касса) | `cash-flow-snapshot` |
| Работа с клиентами | `~~crm` (Битрикс24, amoCRM) | `~~почта` (Яндекс 360) | `crm-maintenance` (демо только на чтение) |
| Найм / вакансии | `~~почта` (Яндекс 360) | `~~календарь` (Яндекс Календарь) | `job-post-builder` |
| Организованность | `~~хранилище` (Яндекс Диск) | `~~почта` | Настройка структуры папок |
| Перегруз расписанием | `~~календарь` | `~~почта` | `business-pulse` |
| В целом / не уверен | `~~почта` | `~~бухгалтерия` | `cash-flow-snapshot` |

Если владелец называет коннектор не из таблицы — добавьте его вторым и используйте
рецепт `business-pulse`.

## Выбор рецепта

Запускайте рецепт «показать пользу» сразу после подключения **первого** коннектора
— не ждите второй. Если коннекторы уже активны на старте сессии, запустите
подходящий рецепт под главную боль владельца до начала интервью. Порядок приоритета:

1. `~~бухгалтерия` или `~~платежи` → `cash-flow-snapshot`
2. `~~crm` → `crm-maintenance` (демо «записать заметку», только на чтение)
3. `~~почта` → поиск непрочитанных писем про счета/оплату, показать топ-3
4. `~~календарь` → `business-pulse`
5. Только `~~хранилище` / Рабочий стол → разобрать настройку структуры папок

**Если данных бизнеса не хватает:** если коннектор `~~бухгалтерия` возвращает
статус «не хватает данных профиля» (нет названия компании или сферы), сначала
заполните недостающее, используя название бизнеса из вопроса 1, и только потом
запускайте `cash-flow-snapshot`. Не пропускайте рецепт — сначала соберите данные.

## Профиль владельца — формат хранения

Запишите этот блок в память сессии Cowork под заголовком `## Контекст бизнеса`.
Каждый скилл читает этот раздел по совпадению заголовка. Не переименовывайте
заголовок и не меняйте названия полей.

```markdown
## Контекст бизнеса

- **Бизнес:** <одна фраза — сфера, продукт/услуга>
- **Размер:** <число человек, включая владельца>
- **Главные боли:** <боль 1> · <боль 2> · <боль 3>
- **Подключённые инструменты:** <список активных коннекторов через запятую>
- **Недельный ритм:** <фраза-триггер и день, например «недельная сверка каждый понедельник»>
- **Онбординг пройден:** <ДД.ММ.ГГГГ>
```

Если файл памяти уже есть — добавьте или обновите только раздел `## Контекст
бизнеса`. Не трогайте остальное.
