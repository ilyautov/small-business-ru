# Гид по запросам к коннекторам

Как подтянуть нужные данные из каждого источника для каждого режима. Конкретные сервисы —
только примеры; в логике работаем через категориальные плейсхолдеры `~~бухгалтерия` и
`~~платежи`.

---

## `~~бухгалтерия` — режим авансов по УСН (доходы/расходы)

Подтяни данные за период с 1 января текущего года по последний день последнего завершённого
квартала. Примеры систем: 1С:Бухгалтерия, МойСклад, Контур.Эльба, СБИС, Моё дело.

Ключевые поля:
- `Доходы` (выручка, поступления, учитываемые на УСН)
- `Расходы` (учитываемые расходы — только для объекта «Доходы минус расходы»)
- `Налоговая база` (= доходы для 6%; доходы − расходы для 15%)

Если система возвращает несколько статей доходов/расходов — просуммируй. Нужна итоговая база
по объекту налогообложения.

**УСН считается кассовым методом** — доход признаётся по поступлению денег, расход по оплате.
Если выгрузка по отгрузке (начислению), отметь в результате — бухгалтер подтвердит базу.

---

## `~~бухгалтерия` — режим выплат подрядчикам

Подтяни все выплаты контрагентам за работы и услуги за налоговый период (1 января – 31 декабря).

Источники в учётной системе:
- Карточка счёта 60 (расчёты с поставщиками/подрядчиками) и 76 (прочие расчёты)
- Ведомость по контрагентам, реестр платежей
- Договоры ГПХ и акты выполненных работ

По каждому контрагенту захвати:
- Наименование (ФИО для физлиц, полное имя для ИП/организаций)
- ИНН (ключ для определения статуса: самозанятый / ИП / физлицо)
- Статус контрагента (самозанятый/ИП/физлицо), если зафиксирован
- Сумму выплат за период
- Даты и суммы платежей (для сверки)
- Наличие договора, акта, чека «Мой налог» (для самозанятых)

**Частая проблема:** статус контрагента в учёте часто не проставлен. Если статус не
определяется — подтяни всех контрагентов со значимыми суммами и дай пользователю/бухгалтеру
проставить статус по ИНН. Отметь это в результате.

---

## `~~платежи` (выписка банка/эквайринга) — режим выплат подрядчикам

Подтяни исходящие платежи контрагентам за работы и услуги за период. Примеры источников:
банк-клиент (выписка), ЮKassa, Тинькофф Касса, Сбер Эквайринг.

Ключевые поля:
- Получатель (наименование/ИНН из назначения платежа)
- Сумма по каждому получателю (итог за год)
- Назначение платежа (подтверждает, что это оплата работ/услуг)
- Дата

**Исключи:** поступления, возвраты, переводы на собственные счета, оплату товаров.

Банковская выписка — основной и самый надёжный источник, если учёт неполный. Назначение
платежа часто содержит ИНН и основание (№ договора) — используй для определения статуса.

## Выгрузка CSV/Excel (fallback)

Если коннектор недоступен, попроси пользователя:
1. Выгрузить ОПиУ / отчёт по доходам из `~~бухгалтерии` в CSV/Excel
   (для авансов — доходы и расходы за период)
2. Выгрузить карточку счёта 60/76 или ведомость по контрагентам (для выплат подрядчикам)
3. Выгрузить выписку из банк-клиента за период (для сверки выплат)

При чтении выгрузок ищи столбцы (названия зависят от системы):
- Доходы/расходы: `Статья`, `Сумма`, `Тип` (Доход / Расход), `Дата`
- Контрагенты: `Контрагент`, `ИНН`, `Сумма`, `Дата`, `Назначение`/`Основание`
- Выписка банка: `Получатель`, `ИНН`, `Сумма`, `Дата`, `Назначение платежа`

Если столбцы не совпадают, попроси пользователя указать столбцы с наименованием получателя,
ИНН и суммой.

---

## Проверка статуса контрагента по ИНН

Статус самозанятого можно проверить на сайте ФНС (сервис «Проверить статус
налогоплательщика НПД» по ИНН и дате). Длина ИНН подсказывает тип:
- 12 цифр — физлицо или ИП (самозанятый — тоже физлицо/ИП с режимом НПД)
- 10 цифр — организация

Точный статус (самозанятый / ИП / обычное физлицо) определяет отчётность — помечай для
проверки бухгалтером, не присваивай автоматически при сомнении.
