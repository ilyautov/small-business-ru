# Ловушки

Известные режимы сбоя скилла business-pulse. Пары «хорошо / плохо».

---

## Ловушка: бухгалтерия вернула неожиданное состояние

**Почему важно:** учётная система может вернуть пустой результат, состояние «синхронизация идёт» или ошибку авторизации без жёсткого исключения. Если принять молчание за «нет данных», пульс тихо пропустит раздел «Деньги», не сказав собственнику — и тот решит, что у его бизнеса нет данных в учёте.

### ✗ Плохо

```
Claude: [бухгалтерия вернула пустой ответ]
        → тихо пропускает раздел «Деньги»
        → пульс без данных по деньгам; собственник думает, что учёт не подключён
```

Собственник 10 минут переподключает учёт, прежде чем поймёт, что он всегда был подключён — просто в переходном состоянии.

### ✓ Хорошо

```
Claude: [бухгалтерия вернула пустой ответ или ошибку]
        → заголовок «Деньги» на месте с пометкой: «н/д — бухгалтерия недоступна (ошибка синхронизации или нужна авторизация)»
        → «Источники недоступны: бухгалтерия — пустой ответ» в приложении
```

Собственник видит пробел явно и сам решает, переподключать или работать с частичным пульсом.

## Ловушка: сбой авторизации почты посреди пульса

**Почему важно:** авторизация почтовых коннекторов бывает нестабильной. Сырая ошибка авторизации посреди пульса выглядит как поломка и подрывает доверие к скиллу.

### ✗ Плохо

```
Claude: [ошибка авторизации почты при сборке]
        → «Ошибка: авторизация почты не удалась. Авторизуйтесь заново.»
        → вывод пульса останавливается или показывает сырую ошибку в «Списке наблюдения»
```

Собственник видит сломанный инструмент, а не полезный бриф.

### ✓ Хорошо

```
Claude: [ошибка авторизации почты]
        → продолжает пульс без раздела «Список наблюдения»
        → приложение: «Почта — ошибка авторизации; Список наблюдения недоступен в этом запуске»
        → остальные разделы не затронуты
```

Пульс всё равно даёт ценность. Собственник информирован, а не встревожен.

## Ловушка: спрашивать разрешения перед тем, как тянуть данные

**Почему важно:** главная ценность скилла — делать работу без переспросов. Собственник, вызвавший пульс, уже неявно одобрил выгрузку. Вопрос «проверить бухгалтерию?» убивает смысл и подрывает доверие к скиллу как к автономному помощнику.

### ✗ Плохо

```
Собственник: «введи в курс по бизнесу»
Claude: «Проверить остаток в бухгалтерии? И можно посмотреть воронку в CRM?»
```

Ещё три круга переписки до первого полезного результата.

### ✓ Хорошо

```
Собственник: «введи в курс по бизнесу»
Claude: [сразу запускает все параллельные вызовы]
        → выдаёт пульс одним ответом
```

## Ловушка: отправка в мессенджер требует явного подтверждения

**Почему важно:** отправка в мессенджер постит в каналы, которые видят другие люди. Автоотправка без подтверждения может поставить собственника в неловкое положение или заспамить команду.

### ✗ Плохо

```
Claude: [в конце пульса]
        «Отправил это в общий канал.»
```

Собственник не просил отправку; теперь вся команда видит финансовые данные.

### ✓ Хорошо

```
Claude: [в конце пульса]
        «Отправить это в мессенджер? Если да — в какой канал?»
```

Отправка происходит только при конкретном «да + название канала» от собственника. Никогда не предполагайте.
