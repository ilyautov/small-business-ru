# Справочник: постановка кампании в очередь через `~~crm`

> Имя файла сохранено как `hubspot-staging.md` для совместимости со структурой
> пака. Содержимое адаптировано под РФ: вместо HubSpot — `~~crm` (**Битрикс24**,
> **amoCRM**, RetailCRM). Эндпоинты ниже — обобщённый шаблон; сверяйте с
> документацией вашей CRM (в Битрикс24 планирование публикаций живёт в модуле
> CRM-маркетинга / мессенджер-маркетинга).

Требуется: тариф `~~crm` с модулем планирования публикаций в соцсетях/мессенджерах.
Авторизация: Bearer-токен (OAuth 2.0 или ключ приложения).

## Проверка тарифа

Если неясно, есть ли у владельца модуль планирования публикаций:

```
GET /crm/v3/objects/companies?limit=1
```

Проверьте тариф/подписку аккаунта. Если API возвращает 403 на эндпоинтах публикаций, скажите владельцу: «Постановка постов в очередь требует тарифа `~~crm` с модулем планирования публикаций. Ваш текущий тариф его не включает. Могу вместо этого выгрузить CSV для планировщика — подойдёт?»

## Создание кампании

```
POST /marketing/v3/campaigns
{
  "name": "Летняя акция 2026 — соцсети",
  "startDate": "2026-06-01",
  "endDate":   "2026-06-30",
  "currencyCode": "RUB",
  "utm": {
    "source": "social",
    "medium": "owned",
    "campaign": "letnyaya-akciya-2026"
  }
}
```

Ответ: `{ "id": "<id_кампании>", ... }` — сохраните для привязки постов.

## Создание поста

Один вызов на строку календаря. Ставьте как `ЗАПЛАНИРОВАНО` (`SCHEDULED`), никогда не `ОПУБЛИКОВАНО` (`PUBLISHED`).

```
POST /marketing/v3/social/posts
{
  "campaignId": "<id_кампании>",
  "channelId":  "<id_канала_в_crm>",
  "content": {
    "body": "<согласованный текст подписи>"
  },
  "scheduledAt": "2026-06-03T10:00:00+03:00",
  "attachments": [
    {
      "url": "<постоянный_png_url_из_дизайна>"
    }
  ],
  "status": "SCHEDULED"
}
```

**`channelId`** — ID подключённого аккаунта соцсети в `~~crm` (не название платформы). Получить подключённые аккаунты:
```
GET /marketing/v3/social/channels
```
Сопоставьте по `type` (`TELEGRAM`, `VK`, `OK`, `DZEN`) и используйте поле `id`.

**`scheduledAt`** — должно быть в будущем (относительно момента вызова API). Используйте время по часовому поясу владельца (МСК = `+03:00`), если он не указал иное.

## Проверка очереди

После постановки всех постов:

```
GET /marketing/v3/social/posts?status=SCHEDULED&campaignId=<id_кампании>
```

Покажите результат таблицей: дата, канал, первые 60 знаков подписи, статус. Дайте прямую ссылку на кампанию:
`https://<ваш-портал>.bitrix24.ru/marketing/campaigns/{id_кампании}`

Скажите владельцу: «Посты в очереди и опубликуются автоматически. Любой пост можно отменить или отредактировать в CRM до времени выхода.»

## CSV-запасной вариант (без модуля планирования)

Когда постановка через `~~crm` недоступна, выгрузите CSV, который владелец импортирует в свой планировщик (SMMplanner, Postoplan, Amplifr и т.п.).

Заголовки столбцов:
```
Дата,Время,Канал,Подпись,СсылкаНаКартинку,Статус
```

Пример строки:
```
03.06.2026,10:00 МСК,Telegram,"Летняя акция: −30% на свечи ☀️ ...",https://.../export/...,Запланировано
```

Скажите владельцу: «Подготовил CSV для планировщика, раз ваш тариф `~~crm` не включает планирование публикаций. Импортируйте его в SMMplanner или Postoplan. [путь к файлу]»

## Справочник полей

| Поле | Тип | Примечания |
|------|-----|------------|
| `campaignId` | строка | UUID из создания кампании |
| `channelId` | строка | Из `GET /social/channels` |
| `content.body` | строка | Текст подписи; до 2 000 знаков |
| `scheduledAt` | ISO 8601 | Время в будущем; со смещением часового пояса (МСК `+03:00`) |
| `attachments[].url` | строка | Публичный URL (URL экспорта из `~~дизайна` подходит) |
| `status` | enum | Всегда `"SCHEDULED"` — никогда `"PUBLISHED"` |
