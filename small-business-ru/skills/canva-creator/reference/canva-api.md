> Имя файла сохранено как `canva-api.md` для совместимости со структурой пака.
> Canva заблокирована в РФ — основной путь это `~~дизайн`: **VistaCreate**
> (быв. Crello), **Supa**, **Поликрафт**, Сберографика. Описанная ниже логика —
> обобщённый шаблон работы с API сервиса дизайна (автозаполнение шаблонов,
> загрузка ассетов, экспорт). Конкретные эндпоинты сверяйте с документацией
> вашего сервиса. Если у владельца есть доступ к Canva — те же шаги применимы
> к Canva Connect API.

Базовый URL: зависит от сервиса (например, VistaCreate API).
Авторизация: Bearer-токен (OAuth 2.0). Нужны права на чтение/запись дизайнов и ассетов.

| Возможность | Бесплатный | Платный | Командный |
|-------------|------------|---------|-----------|
| Создавать дизайны | ✓ | ✓ | ✓ |
| Список своих дизайнов | ✓ | ✓ | ✓ |
| Брендовые шаблоны (чтение) | — | ✓/— | ✓ |
| Автозаполнение брендовых шаблонов | — | ✓/— | ✓ |

> Наличие API-автозаполнения зависит от сервиса и тарифа. Если API недоступно — работайте через ручной выбор шаблона и описание правок для владельца (см. раздел «Копия дизайна»).

## Брендовые шаблоны

**Список шаблонов:**
```
GET /brand-templates?query={ключевое слово}
```
Поля, которые интересны:
- `id` — передаётся в автозаполнение
- `title` — человекочитаемое название
- `thumbnail.url` — превью
- `dataset[].label` — метки полей автозаполнения (например, «Headline», «ProductName»)

Фильтруйте по `title`, содержащему тип ассета (например, «квадратный пост», «сторис», «шапка письма»).

## Автозаполнение

Заполняет переменные поля шаблона и создаёт новый дизайн:

```
POST /autofills
{
  "brand_template_id": "<id_шаблона>",
  "title": "Летняя акция — пост 1",
  "data": {
    "Headline": { "type": "text", "text": "Летняя акция: −30% на свечи" },
    "ProductImage": { "type": "image", "asset_id": "<id_загруженного_ассета>" }
  }
}
```

Ответ: `{ "job": { "id": "<id_задачи>", "status": "queued" } }`

Опрашивайте `GET /autofills/{id_задачи}` до `status == "success"`. В ответе будет `result.design.id` — используйте его для экспорта.

## Копия дизайна (базовый тариф)

На базовых тарифах прямого «копирования шаблона» через API может не быть. Процесс:

1. `GET /designs?query={название шаблона}` — список дизайнов по имени
2. Покажите топ-3 владельцу; получите подтверждение
3. `POST /designs` с указанием типа ассета, чтобы создать пустой дизайн нужных размеров, затем опишите, что владельцу обновить вручную в сервисе.

Генерация на базовом тарифе полуручная: Claude создаёт «заготовку» дизайна и заполняет то, что позволяет API; владелец применяет брендовые правки в сервисе и возвращает ID дизайна для экспорта.

## Загрузка ассета

Используйте, когда бриф ссылается на фото товаров на диске владельца.

**Шаг 1 — инициализация загрузки:**
```
POST /asset-uploads
{ "name_base64": "<base64(имя файла)>", "types": ["image/jpeg"] }
```
Ответ: `{ "job": { "id": "<id_задачи>" }, "upload_url": "<presigned_url>" }`

**Шаг 2 — загрузка файла:**
```
PUT <upload_url>
Content-Type: image/jpeg
Body: <сырые байты файла>
```

**Шаг 3 — опрос до готовности:**
`GET /asset-uploads/{id_задачи}` → дождитесь `status == "success"` → захватите `asset.id`.

Макс. размер файла: 100 МБ. Поддерживаемые типы: `image/jpeg`, `image/png`, `image/webp`.

## Экспорт

**Зачем экспортировать каждый дизайн:** превью из ответа автозаполнения — это
короткоживущий аутентифицированный URL CDN. Он истекает за минуты и
рендерится как битая заглушка «Показать изображение», если встроить его в
markdown-картинку после истечения. Вывод `POST /exports` — **постоянный URL**,
безопасный для встраивания в превью в чате, прикрепления к публикации в
`~~crm` и отправки владельцу.

Всегда экспортируйте каждый дизайн после генерации, до показа превью.

```
POST /exports
{
  "design_id": "<id_дизайна>",
  "format": {
    "type": "png",
    "export_quality": "regular",
    "pages": [1]
  }
}
```

Опрашивайте `GET /exports/{id_задачи}` до `status == "success"`. В ответе `urls[]` — используйте `urls[0]` как ссылку превью и URL вложения для `~~crm`.

**Эквиваленты через MCP** (если используете коннектор `~~дизайна` в Cowork вместо прямого REST):

| Нужно | Инструмент MCP | Возвращает |
|-------|----------------|------------|
| Постоянный URL превью | `export-design` | Постоянный URL загрузки (безопасно встраивать) |
| Превью страницы (стабильнее ответа автозаполнения, но не постоянный) | `get-design-thumbnail` | URL превью страницы |
| Только метаданные дизайна | `get-design` | Название, владелец, число страниц, превью |
| Загрузка с машины владельца | `upload-asset-from-url` | `asset_id` (один URL на вызов) |

**Рекомендации по форматам по каналам:**

| Канал | Тип | Размеры |
|-------|-----|---------|
| Лента Telegram/VK (квадрат) | `png` | 1080×1080 |
| Сторис / вертикаль (VK Клипы, обложка) | `png` | 1080×1920 |
| Лента VK/ОК (горизонталь) | `png` | 1200×630 |
| Шапка письма | `png` | 600×200 |

## Лимиты частоты запросов и бюджет генерации

**Жёсткий лимит:** зависит от сервиса (например, N запросов в минуту на токен). Сверьте с документацией `~~дизайна`.

**Стоимость дизайна:** ~5 вызовов API — 1 `POST /autofills` + 1 `POST /exports` + ~3 раунда опроса (`GET /autofills/{id}`, `GET /exports/{id}`). Частота опроса — 3–5 секунд; более частый опрос жжёт квоту, не ускоряя завершение.

**Безопасный потолок:** ~15–20 дизайнов в минуту оставляют комфортный запас под лимит.

**Рекомендуемый темп (используется Стадией 3):**

```
3 кандидата на строку × 1 строка за раз × пауза 30 с между строками
= 6 дизайнов (~30 вызовов API) за 30 секунд
= 12 дизайнов / минуту, около половины потолка
```

**Формула бюджета на предполёте:**

```
всего_дизайнов = (строк под макет) × (кандидатов на строку, по умолч. 3)
всего_вызовов  ≈ всего_дизайнов × 5
время_генерации ≈ (всего_дизайнов / 12) минут
```

**Паттерн отката при `RATE_LIMIT_EXCEEDED`:**

1. Первый раз за сессию → ждём 60 с, повторяем этого одного кандидата. Трактуем как временный всплеск.
2. Второй раз за ту же сессию, или любая ошибка `quota_exceeded` / дневного лимита → останавливаем генерацию и показываем прогресс владельцу. Не повторяем. Скилл спрашивает владельца: (а) перейти на 1 кандидата на оставшуюся строку, (б) пауза и продолжение через 60 минут, (в) остановиться и перейти к текстам с тем, что уже сгенерировано.

## Коды ошибок

| Код | Значение | Что делать |
|-----|----------|------------|
| `PERMISSION_DENIED` | Нет права или не тот тариф | Проверьте тариф; попросите владельца переподключить `~~дизайн` с нужными правами |
| `DESIGN_NOT_FOUND` | Неверный ID дизайна | Запросите список дизайнов и подтвердите ID |
| `AUTOFILL_FIELD_NOT_FOUND` | Имя поля шаблона не совпадает | Запросите метаданные шаблона и перечитайте `dataset[].label` |
| `RATE_LIMIT_EXCEEDED` | Слишком много запросов | Первый раз: ждём 60 с, повтор. Второй раз: стоп и вопрос владельцу (см. выше) |
| `JOB_FAILED` | Асинхронная задача сбойнула | Проверьте `job.error.message`; частая причина — слишком большой ассет |
