# ЭДО: получение договоров на проверку

Используйте коннектор `~~эдо` (например, Контур.Диадок или СБИС ЭДО), чтобы подтянуть входящие документы, ожидающие подписи.

> Под РФ-сервисы ЭДО публичных MCP-серверов почти нет — большинство работает через ручную выгрузку или собственный коннектор. Имена методов ниже — пример интерфейса; адаптируйте под фактический коннектор. Если коннектора нет — используйте fallback (выгрузка/вставка файла).

## Получить входящие документы

Запросите список документов в статусе «отправлен» / «доставлен» (ожидают действия получателя):

```
Метод: getDocuments (или эквивалент коннектора)
Параметры: { status: "ожидает подписи" }
```

Возвращает список документов с полями: ID документа, тема/название, дата создания, статус, стороны.

## Скачать документ

Запросите полные данные документа по его ID, затем скачайте файл для чтения.

## Чего НЕ делать

- Никогда не вызывайте действия, продвигающие документ по маршруту подписания (подписать, утвердить, отправить дальше).
- Никогда не изменяйте документ через коннектор.
- Никогда не создавайте новый документ — этот скилл работает только на чтение.

## Fallback

Если ЭДО не подключён или документ не найден:

```
«ЭДО не подключён — вставьте текст договора или приложите файл напрямую (PDF/DOCX), либо выгрузите документ из Диадок/СБИС вручную».
```
