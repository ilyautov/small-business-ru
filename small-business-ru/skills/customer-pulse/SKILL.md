# Пульс клиентов

## Быстрый старт

Спросите: *«Что говорят клиенты в этом месяце?»*

Claude подтянет споры по платежам, обращения, переписку по почте и диалоги поддержки за последние 30 дней, сгруппирует их в 3–5 тем с дословными цитатами и выдаст список «сделайте эти 3 вещи на этой неделе».

Чтобы добавить отзывы с Яндекс Карт, 2ГИС, Авито, VK или Отзовика — вставьте их после запуска или скажите «у меня есть отзывы для добавления».

## Рабочий процесс

1. **Задайте период.** По умолчанию: последние 30 дней. Если владелец указал диапазон — используйте его.

2. **Подтяните споры по платежам.** Получите споры/чарджбэки за период из `~~платежей` (например ЮKassa или Тинькофф Касса). Если API возвращает ошибку лимита запросов — пропустите и добавьте `Платежи: лимит запросов — не включено` в раздел «Источники». Не повторяйте, не выдавайте ошибку. Шаблон работы с лимитом — в [reference/gotchas.md](reference/gotchas.md). Если `~~платежи` не подключены — попросите выгрузить операции/споры из банк-клиента в CSV.

3. **Подтяните обращения из CRM.** Получите открытые и недавно закрытые обращения из `~~crm` (например Битрикс24 или amoCRM). Если обращений 0 — запишите `CRM-обращения: 0` и продолжайте, без предупреждения.

4. **Подтяните переписку из почты.** Найдите в `~~почте` (например Яндекс 360) письма за период со словами: `возврат отмена недоволен проблема брак претензия задержка опоздал долго не работает не пришло сломалось`. Возьмите темы писем и по 1–2 предложения из каждого.

5. **Подтяните диалоги поддержки.** Получите открытые и недавно закрытые диалоги из канала поддержки (`~~мессенджер`, например Telegram, или встроенный чат). Берите только сообщения клиентов — исключайте ответы операторов и ботов.

6. **Примите вставленные отзывы (опционально).** Если владелец вставляет текст отзывов с Яндекс Карт, 2ГИС, Авито, VK, Отзовика — включите их в общий пул с тегом `[Отзыв]`. Коннектор не нужен.

7. **Выделите темы.** Сгруппируйте все данные в 3–5 повторяющихся тем. Каждая тема включает:
   - Заголовок в одно предложение (например, «Задержки доставки — повторные жалобы»)
   - 2–3 дословные цитаты с тегом источника: `[Платежи]`, `[CRM]`, `[Почта]`, `[Поддержка]` или `[Отзыв]`
   - Счётчик сигналов (сколько обращений касаются этой темы)

   Дословные цитаты — обязательны: никогда не пересказывайте своими словами. Антипаттерн — в [reference/gotchas.md](reference/gotchas.md).

8. **Составьте список «сделайте эти 3 вещи».** Отсортируйте темы по числу сигналов. Возьмите топ-3 и напишите по одному конкретному шагу под силу владельцу на каждую тему. Формат — нумерованный чек-лист.

9. **Выдайте отчёт.** Структура вывода в таком порядке:
   - **Заголовок** — H2 с «Пульс клиентов» и диапазоном дат.
   - **Источники** — список с числом сигналов на источник (споры по платежам,
     CRM-обращения, письма, диалоги поддержки, вставленные отзывы). Отметьте
     любой источник, который пропущен из-за лимита запросов.
   - **Темы** — для каждой темы жирный нумерованный заголовок с числом
     сигналов, затем две дословные цитаты как цитаты-блоки, каждая с тегом
     источника.
   - **Сделайте эти 3 вещи на этой неделе** — нумерованный список из трёх
     конкретных шагов под силу владельцу, каждый привязан к топ-теме.

   Полный разобранный пример — в [reference/examples/example-report.md](reference/examples/example-report.md).

## Approval gates (контрольные точки)

Этот скилл **только для чтения** — он не публикует, не отправляет, не отвечает и не меняет записи. Подтверждение не требуется.
