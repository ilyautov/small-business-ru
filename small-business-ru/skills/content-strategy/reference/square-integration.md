# Интеграция с источником данных о продажах (будущее улучшение)

> Имя файла сохранено как `square-integration.md` для совместимости со структурой пака. Содержимое адаптировано под РФ: вместо Square (касса/эквайринг) речь о подключении данных о продажах из `~~платежей` (ЮKassa, Тинькофф Касса, СБП) и `~~бухгалтерии` (1С, МойСклад).

Источники данных о продажах перечислены как опциональные коннекторы для content-strategy, но полный путь интеграции для каждого ещё не реализован. Этот документ описывает, что нужно.

## Текущее состояние

- РФ-источники (ЮKassa, 1С, МойСклад) названы в коннекторах, но не до конца подключены к процессу
- Базовый, проверенный путь — **ручная выгрузка операций в CSV/Excel** из банк-клиента, ЮKassa или учётной системы

## Что нужно сделать

### 1. Выбор точки продаж / организации

У бизнеса может быть несколько точек продаж, юрлиц или касс. Перед выгрузкой операций нужно:

1. Запросить список доступных точек/организаций (через API учётной системы или ЮKassa)
2. Спросить владельца: «По какой точке/организации анализировать?» (или по умолчанию основную)
3. Получить идентификатор(ы) для последующих запросов

### 2. Выгрузка операций

Для каждой точки/организации:

```
Запросить операции за период:
- идентификатор точки/организации: <из шага 1>
- начало периода: <90 дней назад, ДД.ММ.ГГГГ>
- конец периода: <сегодня>
- сортировка: по убыванию даты
```

Извлечь:
- Дату операции
- Позиции (наименование товара, количество, цена в ₽)
- Общую выручку по операции

### 3. Категоризация

Операции в кассовых/платёжных системах могут не иметь явной таксономии «товар» / «услуга» — позиции живут в номенклатуре. Сопоставьте позиции операций с номенклатурой в учётной системе, чтобы извлечь названия товаров и категории.

## Почему пока заглушка

1. **Нет тестового аккаунта** — для сквозной проверки нужен реальный аккаунт ЮKassa / 1С / МойСклад
2. **Сложность номенклатуры** — модель номенклатуры в учётных системах гибче выписки по эквайрингу и добавляет поверхность для ошибок
3. **Множественность точек** — выписка из банк-клиента проще; многоточечный бизнес требует от владельца выбрать точку/организацию

## Как реализовать

1. Получить тестовый/демо-аккаунт ЮKassa, 1С или МойСклад
2. Прописать сценарии для одноточечного и многоточечного бизнеса
3. Добавить `reference/sales-data-example.md` с разобранным примером
4. Обновить процесс в SKILL.md, чтобы коннектор стал равноправным путём (не «тоже поддерживается»)
5. Прогнать сценарии с реальным пользователем перед выпуском

## Оценка времени

~4 часа на изучение + реализацию + тестирование при условии доступного тестового аккаунта.
