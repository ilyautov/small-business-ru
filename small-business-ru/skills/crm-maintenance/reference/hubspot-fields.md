# Поля CRM и типы активностей

Конкретные объекты, свойства и типы активностей CRM, из которых скилл `crm-maintenance` читает и в которые пишет. В скоупе только перечисленные здесь поля — всё остальное в CRM остаётся нетронутым.

Имена полей даны как ориентир для Битрикс24/amoCRM — в вашей системе они могут называться чуть иначе; сопоставьте по смыслу.

## Контакты — запись

| Поле (смысл) | Битрикс24 / amoCRM | Использование |
|---|---|---|
| Email | `EMAIL` | Основной идентификатор для поиска + дедупа. Всегда задаётся при создании. |
| Имя | `NAME` | Из подписи письма или приглашения календаря, если доступно. Оставь пустым, если неизвестно. |
| Фамилия | `LAST_NAME` | Из подписи письма или приглашения календаря, если доступно. Оставь пустым, если неизвестно. |
| Телефон | `PHONE` | Из подписи письма, если доступно. |
| Компания | `COMPANY_TITLE` | Из домена email в подписи или организации в календаре, если доступно. |

Не пиши никакие другие свойства контакта. Ответственный, стадия воронки и источник лида — поля, которыми управляет собственник; никогда не перезаписывай.

## Контакты — чтение (для поиска)

| Поле (смысл) | Использование |
|---|---|
| Email | Ключ поиска. Точное совпадение без учёта регистра. |
| Имя · Фамилия · Компания | Показываются собственнику при разрешении неоднозначности. |
| ID объекта | Используется для связывания со сделками и активностями. |

## Сделки — чтение (все пути очистки + поиска)

| Поле (смысл) | Битрикс24 / amoCRM | Использование |
|---|---|---|
| Название сделки | `TITLE` | Показывается собственнику; используется для нечёткого совпадения с темой письма/встречи |
| Стадия сделки | `STAGE_ID` / статус | Только чтение при очистке — помечай расхождения, никогда не меняй |
| Сумма | `OPPORTUNITY` / бюджет (₽) | Чтение при очистке; помечай, если недавнее письмо/встреча подразумевает изменение |
| Дата закрытия | `CLOSEDATE` / дата | Чтение при очистке; помечай, если устарела (формат ДД.ММ.ГГГГ) |
| Следующий шаг | поле «следующий шаг» / задача | Чтение + предложение правок при очистке |
| Ответственный | `ASSIGNED_BY_ID` | Показывается собственнику; никогда не меняется |
| Дата последней активности | `LAST_ACTIVITY` | Используется для определения зависших сделок |
| Связанные контакты | привязки контактов | Используются, чтобы определить, кто из участников недавних писем/встреч есть в сделке |

## Сделки — запись (только путь очистки, с одобрением)

| Поле (смысл) | Правило |
|---|---|
| Следующий шаг | Предлагай правки; пиши только с явного одобрения собственника |
| Дата закрытия | Предлагай правки; пиши только с явного одобрения собственника |
| Сумма (₽) | Предлагай правки; пиши только с явного одобрения собственника |
| Привязка контакта | Предлагай добавить недостающих участников сделки; пиши только с одобрения |

**Никогда не пиши** стадию сделки, воронку, ответственного или любое пользовательское поле при очистке. Ими управляет собственник.

## Активности — запись

| Тип активности | Используется | Задаваемые поля |
|---|---|---|
| Письмо (Email) | Путь письма | Тема (тема переписки), тело (сводка, не вся переписка), метка времени (время последнего сообщения), связанные контакт(ы) + сделка |
| Звонок (Call) | Путь звонка | Заголовок (название события), тело (сводка), длительность (из календаря), метка времени (начало события), связанные контакт(ы) + сделка |
| Примечание (Note) | Путь очистки | Тело примечания (когда помечаешь что-то на будущий разбор, что не ложится в правку поля) |

Используй стандартный словарь активностей вашей CRM. Не выдумывай пользовательские типы активностей.

## Правила связывания

- Каждая активность должна связываться со сделкой И минимум с одним контактом.
- Если контакт создан на лету при занесении активности, свяжи его со сделкой в той же операции, чтобы активность отображалась и в ленте контакта, и в ленте сделки.
- Не связывай контакт со сделкой в потоке занесения активности, если контакт реально не участвовал в этой переписке или встрече.
