Утро уходит на «с кого бы начать», а горячие лиды ждут. Скилл сам сканирует воронку, ранжирует по срочности и потенциалу, подтягивает контекст из переписки и выдаёт готовый список звонков с тезисами — садись и набирай.

Разбери аргументы:
- `--n` (по умолчанию: `5`) — сколько лидов вывести (1–10)
- `--date` (по умолчанию: сегодня) — на какую дату строить список звонков (`ДД.ММ.ГГГГ`)

## Шаг 1 — Скан воронки

По рабочему процессу скилла `lead-triage`:

1. Подтяни открытые сделки и контакты из `~~crm` (например, Битрикс24 или amoCRM) с активностью за последние 30 дней.
2. Подтяни переписку из `~~почта` (например, Яндекс 360) по каждому лиду (последние 3 письма на контакт).
3. Оцени каждый лид по:
   - **Свежесть**: дней с последнего касания собственника (меньше = лучше)
   - **Стадия**: насколько близко к закрытию (более поздняя стадия воронки = выше приоритет)
   - **Сигнал**: любая недавняя входящая активность (открытие письма, ответ, бронь в календаре, визит на сайт)
   - **Сумма**: сумма сделки в ₽ из `~~crm`

## Шаг 2 — Ранжируй и выбери топ N

Отранжируй все оценённые лиды и выбери топ `--n`. При равенстве баллов отдавай приоритет лидам с неотвеченными входящими сигналами.

Для каждого выбранного лида сделай карточку звонка:

```
{Ранг}. {Имя контакта} — {Компания}
Сделка: {сумма} ₽ | Стадия: {стадия} | Последний контакт: {X дней назад}
Сигнал: {последняя активность}

ТЕЗИСЫ
• {тезис из переписки/контекста сделки}
• {тезис из переписки/контекста сделки}
• {открытый вопрос, который стоит задать}

ЦЕЛЬ ЗВОНКА: {одно предложение — продвинуть на следующую стадию / реактивировать / закрыть}
```

## Шаг 3 — Блок в календаре

Для каждого лида из списка предложи забронировать 20 минут в `~~календарь` собственника (например, Яндекс Календарь) на целевую дату.

Покажи предлагаемые записи в календаре:
```
{временной слот} — Звонок: {Имя контакта} ({Компания})
```

Дождись подтверждения собственника, какие звонки бронировать, прежде чем создавать события в календаре.

## Шаг 4 — Черновики follow-up

Для любого лида с неотвеченным письмом старше 3 дней набросай короткий follow-up:
```
Тема: Re: {тема переписки}

Здравствуйте, {имя}!

{Одно предложение со ссылкой на прошлый разговор}. {Одно предложение с понятным следующим шагом или вопросом}.

{Подпись}
```

## Сбои коннекторов

Если `~~crm` недоступна, остановись и скажи собственнику — оценка лидов требует данных CRM. Если ничего не подключено, попроси выгрузить сделки и контакты из CRM в CSV (или вставить вручную). Если `~~почта` недоступна, пропусти Шаги 3–4 (контекст переписки и follow-up) и отметь в выводе: «Почта не подключена — контекст переписки и черновики follow-up пропущены». Если `~~календарь` недоступен, пропусти бронирование и отметь это.

## Approval gates (точки подтверждения)

- **Никогда не отправляй письма автоматически.** Показывай черновики только для подтверждения собственником.
- **Никогда не создавай блоки в календаре без подтверждения собственника** — сначала покажи предлагаемый список.
- **Никогда не меняй стадии сделок в CRM автоматически.**

## Вывод

Покажи отранжированный список звонков с тезисами. Затем покажи предлагаемые блоки в календаре и попроси подтверждения. Затем покажи черновики follow-up и спроси, какие отправлять.
