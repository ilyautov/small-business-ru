# Скоринг лидов в CRM — lead-triage

Маппинг полей CRM (Битрикс24/amoCRM), веса скоринга и дефолты ICP. Имена полей даны как ориентир — в вашей CRM они могут называться чуть иначе; сопоставьте по смыслу.

---

## Какие поля подтянуть

| Поле CRM (смысл) | Битрикс24 / amoCRM | Для чего |
|---|---|---|
| Имя | `NAME` / имя контакта | Отображение |
| Фамилия | `LAST_NAME` / фамилия | Отображение |
| Компания | `COMPANY_TITLE` / название компании | Отображение + матч по ICP |
| Email / телефон | `EMAIL`, `PHONE` / контакты | Получатель follow-up, набор |
| Стадия воронки | `STAGE_ID` / статус сделки | Фильтр (оставить «Лид», «Квалифицированный лид») |
| Статус лида | `STATUS_ID` / поле статуса | Фильтр (исключить «Не квалифицирован») |
| Отрасль | `INDUSTRY` / сфера деятельности | Соответствие ICP |
| Размер компании | `EMPLOYEES` / число сотрудников | Соответствие ICP. Часто пусто; если пусто — считай неизвестным и оценивай соответственно. |
| Источник лида | `SOURCE_ID` / источник | Контекст касания (Авито / Яндекс Директ / VK / сайт / звонок / рекомендация) |
| Дата создания | `DATE_CREATE` / создан | Срочность — возраст лида |
| Дата последней активности | `LAST_ACTIVITY` / последнее касание | Штраф за свежесть |
| Дата примечания | дата последнего комментария | Срочность — свежесть заметки |
| Ответ на письмо | факт ответа в переписке | Вовлечённость — сигнал ответа |
| Открытия письма | счётчик открытий рассылки | Вовлечённость — открытия (только за 30 дней) |
| Визит на сайт | метка визита (Яндекс Метрика / CRM-трекер) | Вовлечённость — визит на сайт |
| Число касаний | количество исходящих | Вовлечённость — объём касаний |

> Источники лидов под РФ, которые встретятся в поле «источник»: **Авито, Яндекс Директ, VK, сайт, входящие звонки, рекомендации**.

## Модель скоринга (совокупный балл 0–100)

Четыре измерения, каждое 0–25. Сумма = совокупный балл.

### Вовлечённость (0–25)
Считай только сигналы за последние 30 дней. Более старые сигналы = 0.

| Сигнал | Баллы |
|---|---|
| Ответ на письмо за последние 14 дней | +15 |
| Открытие письма за последние 7 дней (без ответа) | +8 |
| Визит на сайт за последние 7 дней | +5 |
| >3 исходящих касаний без ответа | −5 |

### Соответствие ICP (0–25)
Дефолтный ICP, если собственник не задал свой: любая отрасль, 1–50 сотрудников.

| Совпадение | Баллы |
|---|---|
| Отрасль + размер оба попадают в ICP | 25 |
| Размер попадает, отрасль неизвестна | 15 |
| Отрасль попадает, размер неизвестен | 12 |
| Ни то ни другое или оба неизвестны | 5 |

### Срочность (0–25)

| Сигнал | Баллы |
|---|---|
| Примечание содержит «срочно», «горит», «дедлайн», «бюджет согласован» | +15 |
| Возраст лида 7–21 день (лучшее окно для follow-up) | +10 |
| Возраст лида <7 дней | +5 |
| Возраст лида >60 дней | −5 |

### Штраф за свежесть (вычитается из совокупного балла)

| Последняя активность | Вычесть |
|---|---|
| <24 часов назад | −25 |
| 1–3 дня назад | −10 |
| 4–7 дней назад | −5 |
| >7 дней назад | 0 |

---

## Свой ICP (переопределение на лету)

Если собственник назвал ICP в моменте («фокус на услугах, 10–100 сотрудников»), применяй его только для этой сессии.
