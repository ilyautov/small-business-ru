# Краевые случаи — lead-triage

---

## Краевой случай: голый триггер «воронка» запускает полный скоринг, когда собственнику нужен просто статус

**Почему важно:** «Воронка» сама по себе неоднозначна — собственнику могут быть нужны суммы по стадиям, а не оценённый список звонков.

### ✗ Плохо
```
Собственник: "как там воронка?"
Claude: [запускает полный скоринг, возвращает список из 12 лидов]
```

### ✓ Хорошо
```
Собственник: "как там воронка?"
Claude: "Быстрый обзор воронки (стадии сделок + общая сумма в ₽) или
         приоритизированный список звонков на сегодня?"
```

## Краевой случай: устаревшие данные вовлечённости завышают баллы

**Почему важно:** счётчик открытий писем в CRM накопительный — открытия из годовой давности рассылки делают холодный лид «горячим».

### ✗ Плохо
```
У лида 20 открытий (все 11 месяцев назад). Получает 25/25 по вовлечённости.
Выходит на 1-е место. Собственник звонит; лид не помнит бренд.
```

### ✓ Хорошо
```
Ограничь сигналы вовлечённости 30 днями. Если все сигналы старше 30 дней,
балл вовлечённости = 0, а в тезисе:
"Сигналы вовлечённости устарели (последний: [дата]) — заходи как с
холодным контактом."
```

## Краевой случай: клиенты попадают в список лидов

**Почему важно:** позвонить «Клиенту» или «Постоянному клиенту» как потенциальному лиду — стыдно.

### ✗ Плохо
```
Фильтр по стадии воронки не применён → действующий клиент выходит на 2-е
место списка.
```

### ✓ Хорошо
```
Фильтруй строго: стадия воронки = «Лид» или «Квалифицированный лид».
Если у контакта стадия не заполнена, включай с пометкой-флагом:
"⚠ Стадия не задана — подтвердите, что это лид, прежде чем звонить."
```

## Краевой случай: предложение слотов, конфликтующих с занятыми событиями

**Почему важно:** предложить время, на которое собственник уже занят, мгновенно подрывает доверие.

### ✗ Плохо
```
Claude предлагает "Вт 14:00–14:30" без проверки календаря.
У собственника в этом слоте уже звонок с клиентом.
```

### ✓ Хорошо
```
Подтяни ~~календарь на ближайшие два рабочих дня, прежде чем предлагать
любой слот. Предлагай только окна без событий с буфером ±15 минут.
Если свободного окна нет, скажи об этом и предложи посмотреть дальше.
```
