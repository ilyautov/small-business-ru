# Подготовка к налогам (быстрый запуск)

Перед налоговым сроком вопрос один: сколько отложить и что собрать, чтобы не
ловить пени и не дёргать бухгалтера в последний день. Эта команда сразу, без
лишних вопросов, считает аванс по УСН (или собирает выплаты подрядчикам / взносы
ИП) и отдаёт готовый пакет бухгалтеру.

Запусти налоговый workflow через скилл `tax-season-organizer`. Действуй сразу — пользователь набрал /tax-prep, фазу discovery пропускаем.

Разбери аргументы:
- `--mode` (по умолчанию выводится из даты — 1–3 кв. → `авансы-усн`, конец года / декабрь → `оба`) — `авансы-усн` для расчёта авансового платежа по УСН, `подрядчики-гпх` для учёта выплат подрядчикам и самозанятым, `взносы-ип` для расчёта страховых взносов ИП «за себя», `оба` для комбинированного прогона (авансы + подрядчики).
- `--year` (по умолчанию — текущий год)

**Рамка:** Открывай каждый результат строкой «Подготовлено для проверки вашим бухгалтером. Не является налоговой консультацией.»

## Шаг 1 — Определи режим

Если `--mode` не передан:
1. Проверь текущую дату. Октябрь–январь → по умолчанию `оба`. Иначе → `авансы-усн`.
2. Подтверди у собственника: «Судя по времени года, подготовлю [режим]. Сделать что-то другое?»

## Шаг 2 — Авансовый платёж по УСН (если режим включает `авансы-усн`)

1. Подтяни ОПиУ нарастающим итогом с начала года по последний завершённый квартал из `~~бухгалтерии` (например, 1С:Бухгалтерия или МойСклад).
2. Если коннектор `~~бухгалтерии` не подключён — попроси выгрузить ОПиУ в CSV или вставить ключевые числа (доходы, расходы) вручную.
3. Уточни объект налогообложения: «Доходы» (6%) или «Доходы минус расходы» (15%). Если не знаешь — спроси, это меняет всю логику.
4. Спроси: «Сколько авансов по УСН вы уже заплатили в этом году?»
5. Рассчитай: налоговую базу нарастающим итогом, налог по ставке объекта, для «Д−Р» — проверку минимального налога 1% от доходов, авансовый платёж к доплате за текущий квартал.
6. Явно назови каждое допущение — ставку (регион мог снизить), объект, страховые взносы к зачёту.
7. Выдай оформленный расчёт со сроком уплаты текущего квартала.

## Шаг 3 — Выплаты подрядчикам ГПХ и самозанятым (если режим включает `подрядчики-гпх`)

1. Подтяни выплаты подрядчикам из всех подключённых источников: `~~бухгалтерия` и `~~платежи` (выписка банка/эквайринга).
2. Сгруппируй по получателю по всем источникам. Помечай вероятные дубли для ручной проверки — никогда не объединяй автоматически.
3. Раздели получателей по налоговому статусу: самозанятый (НПД), ИП, физлицо по договору ГПХ. Для каждого собери реквизиты: ИНН, статус, договор, акт.
4. Для самозанятых проверь наличие чека из «Мой налог» по каждой выплате — без чека расход не подтверждён.
5. Для физлиц по ГПХ отметь обязанность работодателя как налогового агента (НДФЛ + страховые взносы, отчётность 6-НДФЛ/РСВ).
6. Выдай список выплат с разбивкой по статусам, недостающими реквизитами/чеками и отчётностью к сдаче.

## Approval gates

- **Не является налоговой консультацией.** Указывай это в шапке каждого вывода.
- **Называй каждое допущение.** Ставку, объект УСН, регион, исключённые расходы — дай бухгалтеру рычаги.
- **Не объединяй получателей автоматически.** Помечай дубли для ручной проверки.
- **Ничего не подавай и не отправляй.** Результат — только подготовительный материал.

### Пример (иллюстративный, данные условные)

**Вход:** `/tax-prep --mode авансы-усн`. Объект «Доходы», доход с начала года
условно 1 800 000 ₽, авансы ещё не платили.

**Что делает скилл:**
- берёт ставку объекта «Доходы» (помечает её как допущение — регион мог снизить)
- считает налог нарастающим итогом и вычитает уплаченные взносы к зачёту
- называет срок уплаты текущего квартала

**Выход (формат):**
> Подготовлено для проверки вашим бухгалтером. Не является налоговой консультацией.
> Доход с начала года: 1 800 000 ₽ (условно). Аванс к доплате за квартал: X ₽
> (условный расчёт). Срок уплаты — ближайшая дата квартала. Допущения: ставка,
> взносы к зачёту, регион.

*(Все суммы и ставки здесь условные, для иллюстрации формата. Реальные значения
берёт бухгалтер.)*

## Output

Заверши чек-листом следующих шагов для бухгалтера: недостающие реквизиты/чеки самозанятых, допущения к проверке, сроки.
